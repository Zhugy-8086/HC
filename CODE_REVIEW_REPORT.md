## 关键问题汇总 (Top 10 Critical)

### 1. hc64_sub 下溢检测整数溢出 Bug
**文件**: `hc64.c:79`
**问题**: `a->v[0] < b->v[0] + borrow` 在边界情况下 `b->v[0] + borrow` 回绕，导致下溢检测失败。
**影响**: 减法运算结果错误，可能导致静默的数值错误。

### 2. shift_right 系列函数未验证 shift 参数
**文件**: `hc8.c:192`, `hc16.c:99`, `hc32.c:96`, `hc64.c:100`
**问题**: `shift >= 字长` 时触发 C 标准未定义行为。
**影响**: 程序行为不可预测，可能崩溃。

### 3. hc8_to_shc8 未初始化 frac.v[5]
**文件**: `hc8.c:212`
**问题**: `memcpy` 只复制 5 字节，第 6 层未初始化。
**影响**: 返回的有符号数包含垃圾值，后续运算出错。

### 4. hpdc_trie 输出缓冲区无边界检查
**文件**: `hpdc_trie.cpp:90-100`, `hpdc_engine.cpp:126-141`
**问题**: `trie_collect_leaves` 和 `trie_collect_with_sig` 不检查输出数组容量。
**影响**: 缓冲区溢出，可能破坏栈数据。

### 5. wta_compete 整数溢出导致无限循环
**文件**: `hpdc_engine.cpp:48-49`
**问题**: `uint8_t j` 在 `n=256` 时溢出归零，循环条件永远为真。
**影响**: 程序挂死或内存损坏。

### 6. dilate/erode added_map 越界写入
**文件**: `hpdc_engine.cpp:161-164`
**问题**: `added_map[512]` 但 `all_ids[j]` 可能 > 511。
**影响**: 栈缓冲区溢出。

### 7. uart_parse 信任未验证的帧长度
**文件**: `hpdc_network.cpp:37-47`
**问题**: `out->len` 直接从帧中读取，未做边界检查。
**影响**: 越界读取，可能崩溃或信息泄露。

### 8. rle_decode 未验证 RLE 项一致性
**文件**: `hpdc_storage.cpp:91-103`
**问题**: 不验证 `prefix_depth + suffix_len == 6`。
**影响**: 缓冲区溢出。

### 9. hpdc_cpp.hpp 调用不存在的 C 函数
**文件**: `hpdc_cpp.hpp:204-455`
**问题**: 所有函数调用带 `sgn_` 前缀，但 C API 没有这些函数。
**影响**: C++ 层完全无法链接。

### 10. merkle_init 中 pow2 溢出
**文件**: `hpdc_storage.cpp:106-111`
**问题**: `uint16_t pow2` 在 `n_leaves > 32768` 时溢出，导致无限循环。
**影响**: 嵌入式环境中看门狗超时重启。

## 架构建议

### 1. 类型安全改进
- `dc.h:51` 的 `hc8_to_dc` 参数应改为 `const hc8_t*`
- 全局常量应添加 `const` 限定
- 使用 `SGN_` 前缀统一命名 (`precision_t` → `sgn_precision_t`)

### 2. 内存安全改进
- 所有公开函数添加 NULL 指针检查
- 输出缓冲区添加容量参数
- 使用 `#pragma pack(push, 1)` 替代 `#pragma pack(1)`

### 3. API 一致性改进
- 统一 HC8/HC16/HC32/HC64 的函数覆盖
- 统一指针传递方式
- 统一错误处理机制

### 4. 测试改进
- 按模块拆分测试文件
- 补充边界条件测试
- 添加 NaN/Inf/负数输入测试
- 为高级模块添加测试覆盖

### 5. 文档改进
- 修复 hpdc_engine.h 编码损坏
- 为所有公开 API 添加 Doxygen 注释
- 为 Python 绑定添加 docstring

## 总结

HC/SGN 项目是一个设计精巧的超度量数值计算库，架构清晰，模块化良好。主要问题集中在：

1. **边界条件处理不足** - 多个函数缺少参数验证和边界检查
2. **C/C++ 层脱节** - hpdc_cpp.hpp 与 C API 严重不匹配
3. **测试覆盖不均** - 核心模块有测试，高级模块完全无测试
4. **编码损坏** - hpdc_engine.h 注释全部乱码
5. **内存安全** - 多个缓冲区操作缺少边界检查

建议按上述优先级分阶段修复，首先处理 CRITICAL 级别的问题，确保系统稳定性和安全性。
