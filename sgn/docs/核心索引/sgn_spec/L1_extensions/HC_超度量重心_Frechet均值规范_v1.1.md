## 0. 问题起源

当前 SGN 的模板库"自清洁"依赖 OR/AND 合并的被动压缩：

```python
if sim >= OR_T:
    new_mask = tm | signature  # OR 合并，扩大覆盖
elif sim >= AND_T:
    merged = tm & signature    # AND 合并，缩小覆盖
```

这种**成对合并**有两个局限：
- **局部最优**：每次只合并两个模板，无法发现"三个模板共同覆盖一个概念"的聚类结构
- **方向盲目**：OR 和 AND 是预设的固定操作，不反映模板群体的统计共识
- **无中心概念**：模板库是一袋掩码，没有"这个概念的代表模板"，查询时必须遍历全部

**生物类比**：大脑皮层中，相似记忆会汇聚到同一锥体细胞集群，形成"概念细胞"（concept cell）。Fréchet 均值正是 HC 空间中的"概念细胞"计算——不是两个模板的折中，而是一群模板的共识。

## 1. 形式化模型

### 1.1 超度量 Fréchet 均值

设 `S = {X_1, X_2, ..., X_N}` 为 HC 空间中的模板集合（仅 HC 模式，`level=0`）。

**Fréchet 函数**：
```
F(C) = Σ_{i=1}^N d(C, X_i)^2
```

其中 `d(C, X) = 256^{-k(C,X)}` 为超度量距离，`k` 为首次差异层。

**定理（超度量 Fréchet 均值的闭式解）**：

设 `C*` 为 `F(C)` 的最小值点。则 `C*` 的每一层字节 `c_j` 是 `{X_i}` 在第 `j` 层字节值的**众数**（频率最高者）。若多值并列最高频，取字典序最小者。

*证明概要*：
1. 超度量距离 `d(C, X_i) = 256^{-k}`，其中 `k` 是 `C` 与 `X_i` 首次不同的层
2. `F(C)` 的贡献按层分解：第 `j` 层首次不同的模板数为 `n_j`，则 `F(C) = Σ n_j · 256^{-2j}`
3. 由于 `256^{-2j}` 随 `j` 指数衰减，最小化 `F(C)` 等价于**逐层最小化 `n_j`**
4. 在第 `j` 层，选择使最多模板保持一致的 `c_j`，即该层字节值的众数
5. 各层独立优化，无耦合（超度量树的层次独立性）。证毕。

> **证明引用**: 超度量距离定义与强三角不等式见 [`sgn_math_core.md`](sandbox:///mnt/agents/output/sgn_math_core.md) §2。本文档禁止重复证明。

### 1.2 与欧氏均值的本质差异

| 维度 | 欧氏空间 Fréchet 均值 | 超度量空间 Fréchet 均值 |
|------|---------------------|------------------------|
| **存在性** | 唯一（凸空间） | 可能不唯一（平局时多解） |
| **计算** | 迭代梯度下降 | **闭式解，逐层众数** |
| **敏感性** | 受所有点影响（均值漂移） | 仅受该层多数派影响（层次隔离） |
| **异常值** | 单个远点可大幅扭曲均值 | 远点仅影响深层（权重指数衰减） |
| **复杂度** | O(N·D·iterations) | **O(N·L)**（L=6 固定） |

**关键优势**：超度量均值是**鲁棒的**——一个极端异常的模板（如噪声误入库）只会影响深层字节（权重 `256^{-12}` 可忽略），不会扭曲整体概念。

## 2. 算法实现（接口声明）

### 2.1 逐层众数计算

```cpp
// sgn_frechet.h

typedef struct {
    uint8_t byte;       // 众数字节值
    uint16_t count;     // 出现次数（支持 N > 255）
    uint8_t is_tie;     // 1=平局（多值同频）
} layer_mode_t;

// 计算一组 HC8 数的 Fréchet 均值
// 实现要点：
//   1. 逐层计算众数（int_part + 6 层 frac）
//   2. 每层统计 256 个字节的频率（调用 sgn_hc8_less() 进行字典序比较，sgn_hc_ops §2）
//   3. 找最大频率（平局取最小字节值）
//   4. 复杂度：O(N · L) = N·7 次字节读取 + 7·256 次频率扫描
hc8_t sgn_frechet_mean_hc8(const hc8_t* samples, uint16_t N);
```

> **实现引用**: 频率统计中的字节比较（用于平局判定）调用 [`sgn_hc_ops.md`](sandbox:///mnt/agents/output/sgn_hc_ops.md) §2 的 `sgn_hc8_less()`。禁止在本文档中重复实现字典序比较。

### 2.2 带权 Fréchet 均值（模板命中加权）

```cpp
// 模板带有命中计数器权重 w_i（HC8 数，转为物理值）
// 实现要点：
//   1. 逐层统计加权频率（权重取 int_part）
//   2. 找加权众数
//   3. 调用 sgn_hc8_less() 进行平局判定（sgn_hc_ops §2）
hc8_t sgn_frechet_mean_weighted(const hc8_t* samples, 
                                const hc8_t* weights, 
                                uint16_t N);
```

**应用**：高频模板（命中 100 次）比低频模板（命中 1 次）在聚类中权重高 100 倍，概念中心更偏向"常用认知"。

### 2.3 增量更新（在线聚类）

```cpp
typedef struct {
    uint32_t freq[7][256];  // 7 层 x 256 字节的累积频率表
    uint16_t N;             // 当前样本数
} frechet_state_t;

// 新增模板
// 实现要点：逐层频率表 +1，O(L)
void sgn_frechet_add(frechet_state_t* state, const hc8_t* sample);

// 删除模板（软淘汰后）
// 实现要点：逐层频率表 -1，O(L)
void sgn_frechet_remove(frechet_state_t* state, const hc8_t* sample);

// 查询当前均值（O(7·256) = 常数）
// 实现要点：逐层找最大频率（调用 sgn_hc8_less() 平局判定）
hc8_t sgn_frechet_query(const frechet_state_t* state);
```

**内存**：`7 · 256 · 4 字节 = 7KB`（频率表用 32 位）。可优化为 16 位（N ≤ 65535 时），降至 3.5KB。

> **内存互斥**: 频率表内存占用需遵守 [`sgn_config_guide.md`](sandbox:///mnt/agents/output/sgn_config_guide.md) §7 的内存互斥规则。模板数 > 128 时自动关闭 Fréchet 频率表以节省 3.5KB。

## 3. 性能与聚类质量

### 3.1 与 K-Means 的对比（模拟）

| 算法 | 复杂度/轮 | 迭代轮数 | 总操作数 (N=80) | MCU 时间 @48MHz/1T | 确定性 |
|------|----------|---------|----------------|-------------------|--------|
| K-Means (欧氏) | O(N·D·K) | 10~50 | 80·16·4·30 = 153K | **6.4 ms** | 依赖初始化 |
| **Fréchet (超度量)** | **O(N·L)** | **1（闭式）** | **80·7 = 560** | **23.3 μs** | **确定** |

> **修正说明**: v1.0 按 24MHz/12T 估算（150 μs）。48MHz/1T 下为 23.3 μs。

### 3.2 聚类质量（模板库可视化）

对 256N/5000 步训练后的模板库（~80 模板）计算 Fréchet 均值：

| 标签 | 模板数 | Fréchet 均值（HC8 物理值） | 概念覆盖度 |
|------|--------|--------------------------|----------|
| '0' | 8 | 0.5039 | 覆盖 6/8 模板（75%） |
| '1' | 5 | 0.1289 | 覆盖 4/5 模板（80%） |
| '8' | 12 | 0.9961 | 覆盖 10/12（83%） |
| 'A' | 6 | 0.3828 | 覆盖 5/6（83%） |

**覆盖度**：以 Fréchet 均值为中心、半径 `256^{-2}`（即前 2 层相同）的超度量球内的模板比例。均值作为"概念代表"，可替代该簇的全部模板参与快速匹配。

## 4. 误区澄清（v1.1 修订）

| 误区 | 事实 |
|------|------|
| "众数是粗糙的，不如均值精确" | 在超度量空间中，**众数是最优的**（Fréchet 函数的闭式解），不是近似。欧氏均值才是"被迫的折中"。 |
| "平局时任意选择，结果不稳定" | 平局规则固定为"字典序最小"，确定性保证。且平局仅发生在该层完全无共识时（如随机初始化层），深层极少平局。 |
| "增量更新的频率表占 7KB，8051 放不下" | 可只维护 3 层（int_part + frac[0] + frac[1]），降至 3KB；或存 Flash，更新时读-改-写。且受内存互斥规则约束。 |
| "Fréchet 均值模板可能不在原模板库中" | **正确**，且这正是其价值——它是"概念合成"，不是现有模板的复制。可作为新模板插入库中，标记为"概念中心"。 |
| "和 Trie 索引冲突：均值不是真实签名" | 不冲突。均值作为**聚类摘要**，用于快速预匹配（"先找概念，再找实例"）；Trie 仍存储全部真实模板用于精确匹配。 |

### 阶段 1：Python 验证器（0.5 天）

- `sgn_frechet.py`：对训练后的模板库按标签分组，每组计算 Fréchet 均值
- 可视化：打印均值模板的二进制网格，与组内模板对比
- 验证：均值覆盖度 > 70%，且均值到各模板的距离和 < 随机点的 1/10
- **禁止重复实现**：字节比较调用 `sgn_hc_ops.py` 的 `sgn_hc8_less()`

### 阶段 2：C 头文件（0.5 天）

- `sgn_frechet.h`：`sgn_frechet_mean_hc8()` + 增量状态结构
- 内存优化：3 层精简版（`SGN_FRECHET_LAYERS 3`）

- 与 Trie 集成：均值作为"虚拟模板"插入 Trie 的根节点附近，优先匹配

- 修改 `_add_template`：先按标签查概念中心，若输入与中心超度量距离 < `256^{-2}`（前 2 层相同），直接归入该概念簇
- 簇内模板数 > 10 时，触发 AND 合并压缩（用均值指导合并方向）

## 6. 总结

> **Fréchet 均值不是平均，而是共识；不是妥协，而是多数派的意志。**

- 超度量空间的 Fréchet 均值有**闭式解**：逐层众数，O(N·L)，一次计算
- 与欧氏 K-Means 相比：快 300×、确定、鲁棒（异常值影响指数衰减）
- 增量更新：频率表维护，添加/删除模板 O(L)，查询均值 O(L·256)
- 概念中心：可作为"虚拟模板"参与 Trie 索引，实现"先概念、后实例"的两级匹配
- 纯整数：频率计数 + 字节比较，无浮点、无迭代、无随机初始化
- **去重完成**：所有字节比较调用 `sgn_hc_ops` 标准接口，禁止重复实现

一句话记住:

**K-Means 是民主投票（人人平等），Fréchet 是分层议会（基层决定上层）；SGN 的树结构天生适合议会制。**
