## 0. 问题起源

SGN 当前核心通过**交叉相乘**回避除法：

```python
lhs = marked_sum * uc
rhs = unmarked_sum * mc
```

这是线性比例的巧妙技巧，但无法处理**非线性归一化**。当 SGN 从"先验机"向"原型机"进化时，以下场景将触发开方需求：

| 场景 | 为什么需要开方 | 当前规避方案 |
|------|--------------|-----------|
| **自适应学习率冷却** | 经典衰减律 `lr_t = lr_0 / sqrt(t)`，步数 t 增大时学习率按平方根反比下降 | 无，当前 lr 为固定 DiscreteCoordinate |
| **几何距离归一化** | 8x8 网格上像素的空间距离 `sqrt(dx^2 + dy^2)`，用于加权匹配 | 当前仅用汉明距离（popcount），无几何权重 |
| **势能竞争模型** | 响应速度 = base + match*gamma - 势能衰减，平衡态求解涉及开方 | 当前纯线性竞争 |
| **能量均分判定** | 多模板能量分配时，均方根（RMS）作为归一化基准 | 当前无能量概念 |

**关键约束**：若此时被迫引入浮点 `math.sqrt`，SGN 的跨平台确定性（md5sum 一致）和 MCU 无 FPU 运行能力将同时崩塌。

## 1. 形式化模型

### 1.1 2-adic 数与 HC 的同构

HC8（6 层 x 8 位）的仅 HC 模式（`level=0`）与模 `2^48` 的 2-adic 整数截断同构：

```
X = int_part + frac[0]/256 + frac[1]/256^2 + ... + frac[5]/256^6
  ≡ x_0 + x_1*2^8 + x_2*2^16 + ... + x_5*2^40   (mod 2^48)
```

其中 `x_i = frac[i]`（当 i≥1），`x_0 = int_part`。

### 1.2 Hensel 引理（平方根版本）

**定理**：设 `a` 为 2-adic 整数，且 `a ≡ 1 (mod 8)`（即 HC 表示中最低 3 位为 `001`）。则存在唯一的 2-adic 整数 `x`，满足：

```
x^2 ≡ a  (mod 2^n)  对所有 n ≥ 3
x ≡ 1  (mod 4)
```

**Hensel 迭代公式**（牛顿法在 2-adic 域）：

```
x_{k+1} = x_k - (x_k^2 - a) / (2*x_k)
```

在 2-adic 绝对值下，`|2*x_k|_2 = 1/2`（因 `x_k` 为奇数），故分母可逆，迭代收敛速度为**二次**（每步有效位数翻倍）。

> **实现注意**：在 2-adic 整数环 `Z_2` 中，2 不是可逆元。上述公式成立的前提是 `(x_k^2 - a)` 能被 `2*x_k` 整除足够多次，使得商仍是 2-adic 整数。对于软件实现，**逐位提升算法**（见 2.2 节）是更安全、更直接的替代方案。

### 1.3 与 HC 精度层的对齐（v1.2 修正）

Hensel 提升在 2-adic 域中是**逐位（bit）**进行的：从 `mod 2^3` 开始，每步将精度翻倍（`2^k → 2^{k+1}`）。由于 HC 采用 256 进制（每 byte 8 bit），**每填满一个 HC byte 层需要 8 次逐位提升**。

| 迭代步 k | 模数 | 有效精度 | 对应 HC 进度 | 填充位宽 |
|---------|------|---------|------------|---------|
| 0 | 2^3 = 8 | 3 位 | — | — |
| 1 | 2^4 = 16 | 4 位 | — | +1 bit |
| ... | ... | ... | ... | ... |
| 5 | 2^8 = 256 | 8 位 | **int_part 完成** | 第 1 byte |
| 6 | 2^9 = 512 | 9 位 | — | +1 bit |
| ... | ... | ... | ... | ... |
| 13 | 2^16 = 65536 | 16 位 | **frac[0] 完成** | 第 2 byte |
| ... | ... | ... | ... | ... |
| 45 | 2^48 | 48 位 | **frac[5] 完成** | 第 6 byte |
| 47 | 2^48（饱和） | 48 位 | HC8 满精度 | — |

**关键性质**：
- 从 `mod 8` 提升到 `mod 2^48` 需要 **45 步逐位迭代**（而非 6 步）。
- 每 8 步填满一个 HC byte 层，与编译期精度档位 `SGN_HPDC_LEVEL`（2/4/6 层）间接对齐：
  - `LEVEL=2`（16 位小数）：需约 13 步迭代
  - `LEVEL=4`（32 位小数）：需约 29 步迭代
  - `LEVEL=6`（48 位小数）：需约 45 步迭代

## 2. 算法实现

### 2.1 预条件检查（模 8 判定）

```cpp
bool is_quadratic_residue_mod8(const hc8_t& a) {
    // a ≡ 1 (mod 8)  <=>  a 的最低 3 位为 001
    // HC8 中: int_part 的低 3 位 + frac[0] 的高 5 位构成前 8 位
    uint8_t low3 = a.int_part & 0x07;
    return low3 == 0x01;
}
```

**非 1 (mod 8) 的处理**：
- 若 `a ≡ 0 (mod 4)`：提取因子 `4^k`，对剩余部分开方后乘以 `2^k`
- 若 `a` 为完全平方数的 4 倍：如 `a = 4*b^2`，则 `sqrt(a) = 2*b`
- 一般情况：SGN 场景中只处理 `a ≡ 1 (mod 8)` 的预条件输入（如学习率分母、归一化因子）

### 2.2 逐位提升算法（v1.2 推荐实现）

对于 `a ≡ 1 (mod 8)`，存在更直接的逐位提升算法，无需显式求逆。实现时调用 `sgn_hc_ops` 的标准类型转换与比较接口：

```cpp
// 输入: a 为 HC8 数，满足 a ≡ 1 (mod 8)
// 输出: x 为 HC8 数，满足 x^2 ≡ a (mod 2^48)，x ≡ 1 (mod 4)
// 实现要点:
//   1. 将 a 展平为 48 位整数（调用 sgn_hc_ops 类型转换）
//   2. 从 bit 3 到 bit 47 逐位搜索，每步验证 (x_try)^2 ≡ a (mod 2^{bit+1})
//   3. 将结果压缩回 HC8（调用 sgn_hc_ops 逆转换）
// 
// 复杂度: O(有效位数) = 48 次试探，约 96 次 64 位乘法
// 在 48MHz/1T 下约 93.8 μs（参见 sgn_index_matrix §5 性能基准）
hc8_t sqrt_2adic_hc8(const hc8_t& a);
```

> **实现引用**: 具体的 HC8↔uint64_t 转换、逐位比较逻辑见 [`sgn_hc_ops.md`](sandbox:///mnt/agents/output/sgn_hc_ops.md) §2/§3。禁止在本文档中重复实现。

### 2.3 逐字节搜索算法（文档保留方案）

```cpp
// 逐层提升平方根：从最低字节开始，每层求解 byte
// 注意：此处的"层"是 256 进制 byte 层，每步直接填满 8 bit
// 
// 数学等价性: 仅当输入满足 a ≡ 1 (mod 8) 时，逐字节搜索（6 步）
//            与逐位提升（45 步）在模 2^48 下等价。
//            若输入不满足预条件，搜索可能失败。
// 
// 实现要点:
//   1. 初始根 x = 1 (mod 8)
//   2. 每层搜索 byte ∈ [0,255]，使得 (x + byte*2^shift)^2 ≡ a (mod 2^{shift+8})
//   3. 调用 sgn_hc8_add_saturated() 进行 HC 加法（见 sgn_hc_ops §3）
//   4. 调用 sgn_hc8_less() 进行字典序比较（见 sgn_hc_ops §2）
hc8_t sqrt_by_layers(const hc8_t& a);
```

> **v1.2 说明**：`sqrt_by_layers` 在数学上等价于逐位提升的 8 步批量版，但**仅当 `a` 在对应模数下确实存在平方根时**才能保证找到解。对于 `a ≡ 1 (mod 8)` 的输入，Hensel 引理保证了解的存在性，因此该算法安全。若输入不满足预条件，可能搜索失败。

## 3. 性能与精度

### 3.1 与浮点 sqrt 的对比

| 维度 | IEEE754 float64 sqrt | HC8 2-adic sqrt |
|------|---------------------|----------------|
| **精度** | 53 位尾数 | 48 位（HC8 满层） |
| **确定性** | 依赖 FPU/库实现，舍入模式可能不同 | **逐位确定**，md5sum 一致 |
| **硬件依赖** | 必须 FPU 或软件浮点库 | **纯整数 ALU**，8 位 MCU 可运行 |
| **收敛速度** | 硬件指令 1~20 周期 | 45 步逐位迭代，每步 O(1) |
| **适用域** | 所有正实数 | 仅 `a ≡ 1 (mod 8)` 的 2-adic 整数 |
| **开销（Python）** | `math.sqrt` 库调用 | 约 200 次整数运算 |
| **开销（C/48MHz/1T）** | 软件浮点 ~5000 周期 | ~3000 周期（整数乘）≈ **62.5 μs** |

### 3.2 SGN 场景下的精度需求

- **学习率冷却**：`lr_t = lr_0 / sqrt(t)`，t 为步数（1~10000）。`sqrt(10000) = 100`，仅需 7 位精度，HC8 完全覆盖。
- **几何距离**：8x8 网格 `dx, dy ∈ [0,7]`，`sqrt(49+49) ≈ 9.9`，仅需 4 位整数 + 4 位小数，HC8 覆盖。
- **能量 RMS**：`sqrt(sum(x^2)/N)`，N=256，`sqrt(65536/256) = 16`，仅需 5 位整数 + 8 位小数，HC8 覆盖。

## 4. 误区澄清（v1.2 修订）

| 误区 | 事实 |
|------|------|
| "开方在 SGN 中现在就要用" | **不是**。当前 SGN 是静态快照/线性竞争，无开方场景。这是**预判储备**，为几何扩展、动力学扩展保留代数闭包。 |
| "2-adic 开方只能处理 ≡1 (mod 8) 的数，太受限" | SGN 的归一化因子、学习率分母、距离权重均可**预处理**为 ≡1 (mod 8) 形式（乘以 4 或移位）。 |
| "Hensel 迭代比浮点 sqrt 慢" | 在**无 FPU** 的 MCU 上，整数 Hensel 与软件浮点库速度相当，但确定性远胜。 |
| "需要负数支持才能做开方" | 单侧（非负）即可。`sqrt(a)` 定义域 `a ≥ 0`，结果 `x ≥ 0`，无需补码负数。 |
| "每步迭代要算除法，破坏整数化" | 2-adic 除法 = **乘以逆元**，逆元通过逐位提升求得，全程整数。**逐位搜索算法更直接，无需显式除法。** |
| "迭代步数 = HC 层数（6 步）" | **错误（E1 修正）**。2-adic 提升是逐位进行，填满 HC8 的 48 位需要 **45 步**。每 8 步对应一个 byte 层。 |

## 6. 总结

> **2-adic 平方根不是今天的工具，而是明天的门票。**

- SGN 当前无需开方，但**整数代数闭包**（加减乘除开方）的完备性决定了体系能否在不引入浮点的前提下扩展几何、动力学、自适应学习
- Hensel 提升与 HC 分层结构**同构**：每 8 步逐位迭代填满一个 byte 层，`level=6` 时 45 步填满 48 位
- 纯整数、确定性、无 FPU，与 SGN 的跨平台 md5sum 一致性完全兼容
- 预条件 `a ≡ 1 (mod 8)` 可通过预处理满足，逐位搜索算法避免显式除法
- 与补码负数协同：负系数用 SHC8 表示，求值自然正确
- **E1 修正**：Hensel 提升步数为 45 步（逐位），而非 6 步（逐层）
