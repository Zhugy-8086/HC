## 0. 问题起源

当前 SGN 的模板库是**一维字节数组的集合**：

```python
# sgn_core.py
templates = [(label, mask, sc, hc) for ...]
```

工程师只能通过打印十六进制或二进制网格来"观察"模板：

```
Template 0: 0.5039 (0x00, [0x80, 0x00, ...])  # 数字 '0'
Template 1: 0.1289 (0x00, [0x20, 0x00, ...])  # 数字 '1'
```

这种表示**缺乏空间直觉**：
- 无法一眼看出"哪些模板属于同一概念簇"
- 无法判断"异常模板离群多远"
- 无法向非技术利益相关者展示 SGN 的"认知地图"

**核心洞察**：超度量空间的层次结构天然对应**嵌套球面**——首次差异层越深，半径越小，两个点越近。将层索引映射为半径、字节值映射为方向角，即可在欧氏空间中绘制超度量树。

## 1. 形式化模型

### 1.1 超度量球面坐标

对 HC8 数 `X`（以参考点 `O = 0` 为原点）：

**半径** `r(X)`：
```
r(X) = 256^{-k(X)}
```
其中 `k(X)` 为 `X` 与 `O` 首次非零的层索引（`k=0` 为 int_part，`k=1` 为 frac[0]，...）。

**方向** `θ(X)`：
```
θ(X) = 2π * b_k / 256   （二维极坐标）
φ(X)  = π * b_k / 256    （三维球坐标俯仰角）
```
其中 `b_k` 为首次非零层的字节值。

**深层微调**：对于 `k` 层以下仍有差异的字节，作为该方向上的"径向微调"：
```
Δr = Σ_{i=k+1}^{L} b_i / 256^{i-k}
```

### 1.2 嵌套球面可视化

| 层级 | 半径 | 球面含义 | 模板分布 |
|------|------|---------|---------|
| k=0 (int_part) | 1.0 | 最大概念分区 | 所有模板按 int_part 分大球 |
| k=1 (frac[0]) | 1/256 = 0.0039 | 子概念分区 | 同 int_part 下按 frac[0] 分中球 |
| k=2 (frac[1]) | 1/65536 = 1.5e-5 | 细节簇 | 同前两层下按 frac[1] 分小球 |
| k>=3 | < 1e-7 | 个体精度 | 几乎重合，视为同一点 |

**可视化效果**：模板库呈现为**洋葱式嵌套球面**，同一父节点下的模板聚集在同一球壳的相邻区域。

### 1.3 距离场（Distance Field）

定义**超度量距离场** `D(Q)`：对查询点 `Q`，计算到模板库中所有点的超度量距离，取最小值：
```
D(Q) = min_{T in Templates} d(Q, T) = min_{T} 256^{-k(Q,T)}
```

**用途**：
- **异常检测**：若 `D(Q) > 阈值`，说明 `Q` 远离所有已知概念，可能是噪声或新类别
- **概念边界**：`D(Q) = 256^{-2}` 的等值面即为"概念边界"
- **密度估计**：模板密集区域的 `D(Q)` 小，稀疏区域大

## 2. 算法实现（接口声明）

### 2.1 HC8 到二维极坐标

```cpp
// sgn_sphere.h

typedef struct {
    float r;      // 半径（这里用 float 仅用于可视化，核心引擎仍用 HC）
    float theta;  // 方向角 [0, 2π)
    float dr;     // 深层微调半径
} polar2d_t;

// 将 HC8 映射为二维极坐标（用于 PC 端可视化）
// 实现要点：
//   1. 找首次非零层 k（调用 sgn_hc8_less() 与零比较，sgn_hc_ops §2）
//   2. 半径 r = 256^{-k}（float 计算，仅可视化）
//   3. 方向角 theta = 2π * bk / 256
//   4. 深层微调（可选）
polar2d_t sgn_hc8_to_polar2d(const hc8_t* x);
```

> **实现说明**: 首次非零层判定可通过与 HC8 零值的字典序比较实现，调用 [`sgn_hc_ops.md`](sandbox:///mnt/agents/output/sgn_hc_ops.md) §2 的 `sgn_hc8_less()`。禁止重复实现比较逻辑。

### 2.2 模板库散点图生成（Python）

```python
# sgn_sphere.py
import math

def template_scatter(templates):
    # 将模板库转换为二维极坐标散点，供 matplotlib/plotly 绘制
    points = []
    for label, mask, sc, hc in templates:
        # 找首次非零层（调用 sgn_hc8_less() 与零比较）
        k = 0
        bk = hc.int_part
        if bk == 0:
            for i, f in enumerate(hc.frac):
                if f != 0:
                    k = i + 1
                    bk = f
                    break

        r = 256 ** (-k)
        theta = 2 * math.pi * bk / 256

        # 转为笛卡尔坐标
        x = r * math.cos(theta)
        y = r * math.sin(theta)
        points.append((x, y, label))

    return points

# 绘制效果：
# - 同一标签的模板聚集在同一角度扇区（因概念相似，首次非零层字节接近）
# - 不同标签的模板分散在不同扇区
# - 异常模板（孤立点）远离所有簇，肉眼可识别
```

### 2.3 距离场查询（异常检测）

```cpp
// 查询点 Q 到模板库的最小超度量距离
// 返回首次差异层 k（k 越大则距离越小，越正常）
// 实现要点：
//   1. 利用 Trie 快速定位最近邻：从根开始逐层匹配 q 的字节
//   2. 调用 sgn_trie_find_child() 逐层查找（sgn_trie_core §2）
//   3. 首次无法匹配时，当前深度即为 k
//   4. 若全匹配（k > 6），返回 k=7（完全相等）
int sgn_distance_field_query(const HCTrieNode* trie_root, const hc8_t* q);

// 异常判定：若最小距离层 k < ANOMALY_THRESHOLD（如 2），则标记为异常
bool sgn_is_anomaly(const HCTrieNode* trie_root, const hc8_t* q);
```

> **实现引用**: `sgn_distance_field_query()` 内部的所有 Trie 节点查找调用 [`sgn_trie_core.md`](sandbox:///mnt/agents/output/sgn_trie_core.md) §2 的 `sgn_trie_find_child()`。禁止在本文档中重复实现 Trie 遍历代码。

## 3. 性能与可视化效果

### 3.1 可视化质量（模拟 80 模板）

| 标签 | 模板数 | 平均半径 r | 角度扇区 | 簇内距离 |
|------|--------|----------|---------|---------|
| '0' | 8 | 0.0039 (k=1) | 0.2~0.4 rad | 256^-3 |
| '1' | 5 | 0.0039 (k=1) | 0.8~1.0 rad | 256^-3 |
| '8' | 12 | 0.0039 (k=1) | 1.5~1.8 rad | 256^-2 |
| 异常 | 2 | 1.0 (k=0) | 分散 | 256^-1 |

**观察**：
- 正常模板集中在 `r = 0.0039`（k=1）的球壳上
- 异常模板在 `r = 1.0`（k=0）的外围，肉眼可识别为离群点
- 同一标签的模板角度聚集，不同标签角度分离

### 3.2 距离场查询速度

| 模板规模 | 线性扫描 | Trie 查询 | 加速比 |
|---------|---------|----------|--------|
| 64 | 64*6 次比较 | 6 次比较 | 64x |
| 256 | 256*6 次比较 | 6 次比较 | 256x |
| 1024 | 1024*6 次比较 | 6 次比较 | 1024x |

**结论**：Trie 结构使距离场查询从 O(N*L) 降为 O(L)，实时异常检测可行。

## 4. 误区澄清（v1.1 修订）

| 误区 | 事实 |
|------|------|
| "用 float 计算半径破坏 SGN 的整数化原则" | **正确**。`r = 256^{-k}` 的 float 仅用于**可视化后端**，核心引擎仍用整数层索引 `k` 比较。 |
| "球面坐标不是等距嵌入，距离会扭曲" | 是。超度量距离与欧氏距离不等价，但球面坐标的目的是**直观展示**，不是精确度量。 |
| "和 HC 等距嵌入欧氏空间规范重复" | 不重复。等距嵌入追求数学保距，球面坐标追求**可视化直观**。前者是分析工具，后者是展示工具。 |
| "异常检测用距离场不如直接看识别率" | 距离场用于**实时预警**：在 WTA 竞争前就发现输入远离所有概念，可提前拒绝或标记。 |
| "所有模板都在 r=0.0039，无法区分" | 同一球壳上的模板用**角度**区分。且深层微调 `dr` 可进一步分离。 |
| "需要三维可视化库，8051 做不到" | 可视化仅在 PC 端进行。8051 只运行核心的 `sgn_distance_field_query()`（整数层索引）。 |

## 6. 总结

> **球面坐标不是给 HC 加几何，而是给工程师加眼睛。**

- 首次差异层 -> 半径 `256^{-k}`，字节值 -> 方向角，超度量树 -> 嵌套球面
- 模板库可视化：同一概念簇聚集在同一球壳扇区，异常模板在外围游离
- 距离场查询：Trie 加速到 O(L)，实时检测远离所有概念的异常输入
- 与 Fréchet 均值协同：均值是簇中心，球面坐标展示簇形状和离群点
- PC 端 float 仅用于绘图，核心引擎仍用整数层索引，8051 零负担
- 为人工审计、教学演示、异常监控、概念验证提供几何直觉
- **去重完成**：所有 Trie 查询调用 `sgn_trie_core` 标准接口，禁止重复实现

一句话记住：

**十六进制是密码，球面图是地图；密码防外人，地图给向导。**
