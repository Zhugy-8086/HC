## 0. 问题起源

当前 SGN 的数值运算仅支持加减法和固定缩放：

```python
# sgn_core.py
base += lr * match  # 线性更新
hit_counter >>= 1   # 固定除以 2
```

但以下场景将触发**幂运算需求**：

| 场景 | 为什么需要幂 | 当前规避 |
|------|------------|---------|
| **自适应学习率衰减** | `lr_t = lr_0 * gamma^t`，步数 t 增大时学习率指数下降 | 固定 lr，无衰减 |
| **指数型增益调度** | `gamma_t = gamma_0 * beta^t`，训练后期降低增益 | 固定 gamma |
| **几何缩放** | 特征空间中的 `2^n` 倍放大/缩小 | 仅线性加减 |
| **复合增长** | 模板命中计数器的指数奖励 `bonus = base * 2^n` | 线性加固定值 |
| **时间常数** | RC 衰减模拟 `V(t) = V_0 * e^{-t/τ}` 的整数化近似 | 无时间常数概念 |

**核心矛盾**：若引入浮点 `math.pow`，SGN 的跨平台确定性和 MCU 无 FPU 能力将崩塌。

## 1. 形式化模型

### 1.1 二进制快速幂

对整数 `n >= 0`，将 `n` 表示为二进制：
```
n = b_k*2^k + b_{k-1}*2^{k-1} + ... + b_1*2 + b_0
```

则：
```
x^n = Π_{i=0}^{k} (x^{2^i})^{b_i}
```

**算法**：
1. 初始化 `result = 1`
2. 当 `n > 0`：
   - 若 `n & 1`：`result = result * x`
   - `x = x * x`（平方）
   - `n >>= 1`
3. 返回 `result`

**复杂度**：`O(log n)` 次乘法，而非 `O(n)` 次。

### 1.2 HC 多精度乘法（接口声明）

两个 HC8 数的乘法需将字节数组视为 256 进制整数，执行卷积：

```
C[k] = (Σ_{i+j=k} A[i]*B[j] + carry) mod 256
```

其中 `A[0]=int_part, A[1..6]=frac[0..5]`，`B` 同理。结果最多 14 字节（7+7），需压缩回 HC8（截断/饱和）。

```cpp
// HC8 多精度乘法
// 实现要点：
//   1. 将 HC8 展平为 7 字节数组（int_part + 6 frac）
//   2. 执行 7x7 字节卷积（带进位），结果 14 字节
//   3. 压缩回 HC8：处理进位链，int_part 饱和 255（仅 HC 模式）
//   4. 调用 sgn_hc8_add_saturated() 处理进位链压缩（sgn_hc_ops §3）
hc8_t sgn_hc8_mul(const hc8_t* a, const hc8_t* b);
```

> **实现引用**: 多精度乘法的进位链与压缩逻辑调用 [`sgn_hc_ops.md`](sandbox:///mnt/agents/output/sgn_hc_ops.md) §3 的 `sgn_hc8_add_saturated()`。禁止在本文档中重复实现带进位加法。

### 1.3 与仅 HC 模式的兼容性

仅 HC 模式下（`level=0`），乘法结果可能溢出 `int_part`（255）。处理策略：
- **饱和**：`int_part` clamp 到 255，适用于有上限物理量
- **截断**：取低 8 位，适用于模运算场景
- **拒绝**：若溢出则报错，强制使用合体模式

**推荐**：快速幂在仅 HC 模式下使用**饱和策略**，因物理量（如学习率）有天然上限。

> **溢出语义引用**: 仅 HC 模式饱和策略详见 [`sgn_config_guide.md`](sandbox:///mnt/agents/output/sgn_config_guide.md) §2。

## 2. 算法实现（接口声明）

### 2.1 快速幂（C++）

```cpp
// 快速幂：x^n，n 为无符号整数
// 实现要点：
//   1. 初始化 result = 1.0（HC8: int_part=1, frac=全零）
//   2. 二进制分解 n：while(n>0) { if(n&1) result = sgn_hc8_mul(result, base); 
//                           base = sgn_hc8_mul(base, base); n >>= 1; }
//   3. 每次乘法调用 sgn_hc8_mul()（见 §1.2），其内部调用 sgn_hc8_add_saturated()
//   4. 复杂度：O(log n) 次乘法
hc8_t sgn_hc8_pow(const hc8_t* x, uint16_t n);
```

> **实现引用**: `sgn_hc8_pow()` 内部的所有乘法调用 `sgn_hc8_mul()`，而 `sgn_hc8_mul()` 的进位链调用 [`sgn_hc_ops.md`](sandbox:///mnt/agents/output/sgn_hc_ops.md) §3 的 `sgn_hc8_add_saturated()`。禁止在本文档中重复实现乘法或加法。

### 2.2 学习率衰减应用（Python 原型）

```python
# sgn_pow.py

def learning_rate_decay(lr0: HPDC8, gamma: HPDC8, t: int) -> HPDC8:
    # lr_t = lr_0 * gamma^t
    # 例如：lr0=0.5, gamma=0.99, t=100
    gamma_pow_t = hc8_pow(gamma, t)  # O(log t)，调用 sgn_hc8_pow()
    return sgn_hc8_mul(lr0, gamma_pow_t)  # 调用 sgn_hc8_mul()

# 对比：若 gamma=0.5, t=10
# gamma^10 = 0.5^10 = 0.0009765625
# HC8 表示：int_part=0, frac[0]=0x01 (1/256 = 0.0039)
# 实际精度：6 层可表示到 2^-48 = 3.5e-15，足够覆盖
```

## 3. 性能与精度

### 3.1 快速幂复杂度（48MHz/1T 修正）

| n 范围 | 二进制位数 | 乘法次数 | HC8 乘法周期 | 总周期 @48MHz/1T | **时间** |
|--------|----------|---------|------------|-----------------|---------|
| 1~255 | 8 | <= 15 | ~200 | ~3,000 | **62.5 μs** |
| 1~4095 | 12 | <= 23 | ~200 | ~4,600 | **95.8 μs** |
| 1~65535 | 16 | <= 31 | ~200 | ~6,200 | **129.2 μs** |

> **修正说明**: v1.0 按 24MHz/12T 估算（125 μs/192 μs/258 μs），严重过时。48MHz/1T 下速度提升 24 倍。

### 3.2 精度损失分析

| 运算 | 理论值 | HC8 结果 | 误差 | 来源 |
|------|--------|---------|------|------|
| 0.5^2 | 0.25 | 0.25 | 0 | 精确 |
| 0.5^3 | 0.125 | 0.125 | 0 | 精确 |
| 0.5^10 | 9.77e-4 | 1/1024 = 9.77e-4 | < 1e-15 | 截断误差 |
| 0.99^100 | 0.366 | ~0.366 | < 1e-12 | 累积舍入 |

**结论**：对于 `gamma` 接近 1 的衰减（如 0.99^100），HC8 的 48 位小数精度足够。对于 `gamma=0.5` 的剧烈衰减，10 步后值已落入 `frac[2]` 层，仍在表示范围内。

## 4. 误区澄清（v1.1 修订）

| 误区 | 事实 |
|------|------|
| "快速幂需要 O(log n) 次乘法，MCU 太慢" | v1.0 按 24MHz/12T 估算已过时。48MHz/1T 下 n<65535 时 **< 130 μs**，可接受。 |
| "乘法溢出后饱和，结果失真严重" | 仅当 `x > 1` 且 n 大时才会饱和。学习率衰减场景 `x < 1`，结果递减，不会溢出。 |
| "和 2-adic 平方根重复：都是非线性" | 不重复。平方根是 `x^(1/2)`（分数次幂），快速幂是 `x^n`（整数次幂）。后者更简单，无需 Hensel。 |
| "负数次幂怎么办（如 x^-1 = 1/x）" | 当前不支持。负数次幂需除法，SGN 当前规避除法。若未来需要，结合 Hensel 求逆元。 |
| "gamma^t 的 t 很大时（如 1e6），精度丢光" | 是。`0.99^1e6` 下溢为 0。但训练步数通常 < 1e5，且 HC8 可表示到 2^-48，足够。 |
| "和线性衰减 lr -= delta 比，幂运算没必要" | 线性衰减是 `O(1/t)`，幂衰减是 `O(gamma^t)`。后者在训练初期衰减慢（保留大步长），后期衰减快（精细调整），更符合优化理论。 |

## 6. 总结

> **快速幂不是让乘法变快，而是让重复乘法变成二进制跳跃。**

- 二进制快速幂：O(log n) 次 HC8 乘法，替代 O(n) 次线性累乘
- HC8 多精度乘法：7 字节 x 7 字节带进位，结果压缩回 HC8（饱和策略）
- 学习率指数衰减：`lr_t = lr_0 * gamma^t`，训练初期大步长、后期精细调整
- 与线性衰减、软阈值形成完整调度体系：线性做粗调，指数做细调，软阈值做剪枝
- 纯整数、确定性、无浮点，与 SGN 跨平台 md5sum 一致性完全兼容
- **去重完成**：所有乘法进位链调用 `sgn_hc_ops` 标准接口，禁止重复实现
- 为复合增长、几何缩放、时间常数等场景预留代数接口

一句话记住：

**线性衰减是走楼梯，指数衰减是坐滑梯；楼梯每一步一样高，滑梯越来越快。**
