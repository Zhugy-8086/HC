## 0. 问题起源

当前 SGN 的数值体系以**非负**为隐含前提：

```python
# sgn_core.py
base = ConfigItem('BASE', 50, int, ...)   # 假设 ≥ 0
lr   = ConfigItem('LEARNING_RATE', 3, int, ...)  # 假设 ≥ 0
```

但以下未来场景将触发**负数需求**：

| 场景 | 为什么需要负数 | 当前规避方案 |
|------|--------------|-----------|
| **梯度差值** | 两个神经元 base 速度相减，`base_a - base_b` 可能为负 | 用交叉相乘回避减法 |
| **反向位移** | 模板在特征空间中的相对偏移，`Δx = x_target - x_current` | 无，当前仅正向 OR/AND |
| **势能竞争** | 响应速度 = base + match*gamma - 势能衰减，势能可超越前两项 | 当前纯线性竞争，无势能 |
| **误差信号** | 预测值与目标值的偏差，`error = pred - target` | 无误差概念 |
| **对称激活** | 正负双向抑制/兴奋，如 `+1` 兴奋、`-1` 抑制 | 当前仅单向兴奋 |

**核心矛盾**：若此时被迫引入 Python `int` 的有符号大整数，SGN 的跨平台确定性（md5sum 一致）和 MCU 无大整数运算能力将同时崩塌。

## 1. 形式化模型

### 1.1 基 256 补码表示

设 HC8 为 `W = 56` 位无符号整数（`int_part + 6×8` 位），其补码表示：

```
[x]_补 = (2^W - |x|) mod 2^W   (x < 0)
[x]_补 = x                       (x ≥ 0)
```

**字节级实现**：
- 对 `|x|` 的每个字节取反（`~b`）
- 最低字节加 1
- 进位向高位传播

**示例**（W=16 位简化，2 字节）：
```
x = +1:  [0x00, 0x01]
x = -1:  [0xFF, 0xFF]   (0x0001 取反 = 0xFFFE，+1 = 0xFFFF)
x = -256:[0xFF, 0x00]   (0x0100 取反 = 0xFEFF，+1 = 0xFF00)
```

### 1.2 有符号 HC8 结构

```cpp
typedef struct {
    uint8_t sign;        // 0 = 正/零，1 = 负（仅用于快速判断，实际比较用补码）
    uint8_t int_part;    // 整数部分（补码解释下可为 0~255）
    uint8_t frac[6];     // 小数层（补码解释）
} shc8_t;  // signed hc8
```

**关键**：`sign` 字段是**冗余提示位**，真正的符号由补码最高位（`int_part` 的 bit7）决定。`sign` 仅用于快速路径优化，不参与比较/加法核心逻辑。

### 1.3 补码下的 HC 比较（修正 v1.2）

**定理（同号半区）**：设 `A, B` 为补码表示的 HC8 数，且 `A, B` 同为非负或同为负。则 `A < B`（有符号）当且仅当 `A < B`（无符号字典序）。

*证明*：补码将负数映射到无符号空间的上半区 `[128, 255]`，正数映射到下半区 `[0, 127]`。在同一半区内，无符号字典序与有符号序一致。∎

**跨号比较规则**：
- 若 `A` 为非负（`int_part ∈ [0, 127]`），`B` 为负（`int_part ∈ [128, 255]`），则 **有符号上 `A > B`**，但 **无符号字典序上 `A < B`**。
- 因此，跨号比较不能直接使用无符号字典序，需先以 `int_part` 的最高位（bit7）为符号位分流，再在各半区内使用字典序。

**工程实现（接口声明）**：

```cpp
// 有符号 HC8 比较
// 实现要点：
//   1. 提取符号位（int_part 的 bit7）
//   2. 异号时：正数 < 负数（有符号语义）
//   3. 同号时：复用 sgn_hc8_less() 字典序比较（sgn_hc_ops §2）
// 复杂度：O(1) 符号判断 + O(L) 字典序
bool sgn_shc8_less(const shc8_t* a, const shc8_t* b);
```

> **实现引用**: 同号半区内的字典序比较直接调用 [`sgn_hc_ops.md`](sandbox:///mnt/agents/output/sgn_hc_ops.md) §2 的 `sgn_hc8_less()`。禁止在本文档中重复实现。

## 2. 算法实现（接口声明）

### 2.1 取负运算

```cpp
// 对 HC8 执行补码取反：x -> -x
// 实现要点：
//   1. 逐字节取反（~b）
//   2. 最低字节加 1（调用 sgn_hc8_add_saturated 或手动进位）
//   3. 进位向高位传播（复用 sgn_hc8_add_saturated 的进位链）
//   4. sign 字段取反（若结果非零）
// 调用接口: sgn_hc8_add_saturated() [sgn_hc_ops §3]
shc8_t sgn_shc8_neg(const shc8_t* x);
```

> **实现引用**: 取负运算本质是「按位取反 + 1」，即与 HC8 全 1 数相加。加法进位链调用 [`sgn_hc_ops.md`](sandbox:///mnt/agents/output/sgn_hc_ops.md) §3 的 `sgn_hc8_add_saturated()`。禁止重复实现带进位加法。

### 2.2 有符号加法（补码加法）

```cpp
// 补码加法 = 无符号带进位加法，结果自然为补码
// 实现要点：
//   1. 复用 sgn_hc8_add_saturated() 的进位链（sgn_hc_ops §3）
//   2. 仅 HC 模式下 int_part 饱和 255，不向 level 进位
//   3. 溢出时按仅 HC 模式安全舱壁处理（sgn_config_guide §2）
//   4. sign 字段由结果最高位自动推导
shc8_t sgn_shc8_add(const shc8_t* a, const shc8_t* b);
```

### 2.3 有符号减法

```cpp

// 实现：sgn_shc8_add(a, sgn_shc8_neg(b))

```

> **实现引用**: 减法复用取负和加法，两者均调用 [`sgn_hc_ops.md`](sandbox:///mnt/agents/output/sgn_hc_ops.md) §3/§4 的标准接口。

## 3. 性能与精度

### 3.1 范围与精度

| 类型 | 正数范围 | 负数范围 | 精度 | 零表示 |
|------|---------|---------|------|--------|
| 无符号 HC8 | [0, 255.999...] | 无 | 48 位小数 | 全零 |
| **补码 SHC8** | **[0, 127.999...]** | **[-128, 0)** | **48 位小数** | **全零** |

**代价**：引入符号位后，`int_part` 的有效正数范围从 255 缩减到 127（因最高位为符号位）。若应用需大量正数 > 127，应使用**合体模式**（level 扩展）而非仅 HC 模式。

### 3.2 MCU 开销（AI8051U，48MHz/1T）

| 操作 | 机器周期 | 48MHz/1T 时间 |
|------|---------|--------------|
| 取负（~ + 1，7 字节） | ~60 | **2.5 μs** |
| 有符号加法（7 字节） | ~50 | **2.1 μs** |
| 有符号减法（取负 + 加） | ~110 | **4.6 μs** |
| 与无符号加法对比 | ~40 | **1.7 μs** |

**结论**：有符号运算开销约为无符号的 1.3~2.5 倍，仍在可接受范围。

## 4. 误区澄清（v1.2 修订）

| 误区 | 事实 |
|------|------|
| "补码负数破坏 HC 的超度量性质" | **不破坏**。补码在同号半区内保持超度量距离；跨号比较需先按符号位分流，分流后各半区内部超度量仍然成立。证明见 [sgn_math_core.md](sandbox:///mnt/agents/output/sgn_math_core.md) §5.1。 |
| "引入负数后，模板库/像素值不能用了" | 模板库和像素值继续使用**无符号仅 HC**，负数仅用于特定场景（梯度差、势能）。两者通过 `sgn_hc8_to_shc8` 安全转换。 |
| "范围减半（127 vs 255）是巨大损失" | 是，但负数场景通常需要对称范围（如 [-128, 127]），而非大正数。若需大正数，用合体模式。 |
| "和 2-adic 平方根冲突：都是扩展运算" | 不冲突。补码是**表示层扩展**（支持负数），2-adic 平方根是**运算层扩展**（支持开方）。两者独立。 |
| "需要修改所有现有比较代码" | 不需要。现有无符号 HC 比较代码完全不变。新增 `shc8_t` 类型和专用比较函数，跨号时先分流再比较。 |
| "补码下 soft_threshold 怎么定义" | 双侧软阈值：`soft(x, Λ) = sign(x)·max(|x| - Λ, 0)`，对补码取绝对值后执行单侧阈值。实现调用 `sgn_hc8_soft_threshold()` [sgn_hc_ops §5]。 |

## 6. 总结

> **补码负数不是让 HC 变复杂，而是让代数变完整。**

- 基 256 补码：按字节取反加一，进位从低位向高位传播，与无符号进位同构
- 有符号比较 = 同号时字典序；跨号时先按符号位分流，再在各半区内字典序比较
- 与仅 HC 模式兼容：正数直接嵌入，负数通过 `shc8_t` 扩展类型支持
- 阿贝尔群闭合：加减法、取负、零元、交换律/结合律全部满足
- 为梯度差值、反向位移、势能竞争、误差信号等未来场景保留代数接口
- 纯整数、确定性、无浮点，8051 可直接执行
- **去重完成**：所有加法/减法/比较运算调用 `sgn_hc_ops` 标准接口，禁止重复实现

一句话记住：

**无符号 HC 是半条命，补码 SHC 是整条命；半条命能跑，整条命能回头。**
