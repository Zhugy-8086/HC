HPDC map 设计不完美之处（正式记录）

版本：基于当前代码（hpdc_cpp.hpp, pysgn.cpp, hpdc_sandbox.cpp）
更新日期：2026-06-13

---

一、语义模糊：投影空间 vs 物理值空间

· 描述：map 实际作用在投影值 φ = physical / R（R = 2^int_bits）上，而非物理值本身。文档未明确强调这一区别，用户容易误解为物理值空间的映射。
· 示例：期望物理值的平方 physical^2 时，必须手动缩放：sb.map(h, lambda x: (x * R)**2 / R)，需要知道 R。
· 不完美点：缺少“物理值空间 map”变体，或文档醒目警告。
· 影响程度：中

---

二、性能缺陷：每次 map 都重复转换

· 描述：map 内部执行 project(h) + fn(…) + inverse(…)。批量逐元素调用时产生大量函数调用和边界检查。
· 现状：虽有 project_batch / inverse_batch，但未提供 map_batch，用户需手动循环，破坏简洁性。
· 不完美点：缺少批量版本，未利用 SIMD 或预分配缓冲区。
· 影响程度：中

---

三、类型固化：输入输出 HC 类型必须相同

· 描述：map 返回类型固定为输入类型，不支持 HC8 → HC16 等类型提升（如累加防止溢出）。
· 不完美点：无类型转换的 map 重载。
· 影响程度：低

---

四、异常处理不统一

· 描述：
  · C++ 层：用户 fn 抛异常直接穿过 map，可能跳过 inverse，破坏沙盒状态（尽管沙盒无状态，但外部变量可能被改）。
  · Python 层：异常被捕获转为 Python 异常，但 project 已调用无法回滚（无副作用，可接受）。
  · 无事务语义：无 try_map 或 map_or 处理失败（如除零返回原值/默认值）。
· 不完美点：缺乏异常安全保证和替代路径。
· 影响程度：低

---

五、数值稳定性隐患：投影/逆投影的精度损失（有替代方案）

· 描述：map 包含 HC → double → HC 往返转换，存在量化误差。多次链式 map（甚至恒等映射）会导致数值漂移。
· 示例：
  ```python
  h = HC8(0.123456789)
  for _ in range(100):
      h = sb.map(h, lambda x: x)   # 恒等映射
  ```
  结果逐渐偏离原值。
· 不完美点：无“原地 map”或“高精度中间累加器”模式。
· 替代方案（用户指正）：使用 DC（十进制定点数） 可完全避免此问题，因为 DC 无往返转换，且与十进制字面量精确对齐。对于需要稳定精度的场景，应优先选择 DC。
· 影响程度：中（有替代方案后降为低）

---

六、对非 [0,1) 映射未做裁剪

· 描述：用户函数返回值若超出 [0,1)（如负数或 >1），inverse 会静默饱和到 0 或 HC_MAX，不发出警告。
· 示例：sb.map(h, lambda x: x - 0.5)，当 φ < 0.5 时结果为 0，可能非用户本意。
· 不完美点：缺少可选的 clip 或 error_on_out_of_range 模式。
· 影响程度：高

---

七、与有符号 HC 的交互不自然

· 描述：map 只支持无符号 HC8/16/32，不支持 sgn_shc8_t。有符号值需手动转换或丢失符号。
· 不完美点：无 map_signed 或自动处理符号的版本。
· 影响程度：低
